# TASK-0217: Mode-aware postcall extraction outputs (customer vs owner)

Status: DONE  
Assigned: Codex Agent A  
Date: 2026-02-19 (America/New_York)

## Objective
Prevent owner-mode post-call extraction from creating customer lead artifacts.

## Required behavior
- `ai_mode=customer`:
  - emits `postcall.summary` (always)
  - may emit `postcall.lead`
  - may emit `postcall.appt_request`
- `ai_mode=owner`:
  - emits `postcall.summary` only
  - never emits `postcall.lead` or `postcall.appt_request`

## Constraints
- Preserve fail-closed behavior (`postcall.extract_failed` + 422 on schema invalid).
- No Flow A hot-path changes.
- No schema/table changes.

## Acceptance coverage
- Customer transcript path writes summary + lead (+ optional appt_request).
- Owner transcript path writes summary only.
- Reconcile compatibility preserved because it forwards ai_mode from `flow_a.call_stopped`.
