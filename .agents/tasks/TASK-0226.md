# TASK-0226: Postcall SMS notifier (admin, out-of-band)

Status: DONE  
Assigned: Codex Agent (Backend / Automation)  
Date: 2026-02-19 (America/New_York)

## Objective
Add admin endpoint to notify owner by SMS for new customer lead/appt artifacts without touching Flow A hot path.

## Endpoint
- `POST /admin/postcall/notify/sms`
- Auth: `Authorization: Bearer <VOZ_ADMIN_API_KEY>`
- Request (strict JSON):
  - `tenant_id`
  - `since_ts`
  - `limit` (default `50`, hard cap `<= 200`)
  - `dry_run` (default `true`)

## Gates
- `VOZ_FEATURE_POSTCALL_NOTIFY_SMS`
- `VOZ_POSTCALL_NOTIFY_SMS_ENABLED`

## Behavior
- Scans tenant events newest-first for `postcall.lead` + `postcall.appt_request`.
- Dedupes by `rid` and skips if `notify.sms_sent` already exists.
- Composes SMS using summary headline and caller number best-effort.
- `dry_run=true` returns planned notifications and writes nothing.
- `dry_run=false` sends Twilio SMS and emits:
  - `notify.sms_sent` (idempotent key `notify_sms:{rid}`)
  - `notify.sms_failed` on send failure.

## Acceptance coverage
- 401 auth enforcement
- 503 gate enforcement
- dry run has no writes
- non-dry run writes `notify.sms_sent` and second run skips duplicates
- tenant isolation enforced
- `limit > 200` returns `422`
